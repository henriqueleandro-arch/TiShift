The Challenge

Enterprises running SQL Server face escalating licensing costs, hard scaling ceilings, and vendor lock-in. Migration is often blocked by T-SQL procedures, CLR assemblies, SQL Agent jobs, linked servers, `INSTEAD OF` triggers, and multi-schema architectures that traditional tooling cannot convert reliably.

The Solution

TiShift provides one migration workflow across assessment, conversion, bulk load, CDC sync, and validation. It inventories source complexity, estimates automation coverage, converts what can be automated, and flags true blockers before teams waste time on poor-fit projects.

What’s Inside TiShift for SQL Server

Deterministic CLI

Handles repeatable rule-based work: schema transformation, collation and type mapping, data loading, sync orchestration, and validation.

AI Skills

Analyze complex stored procedures, convert CLR assemblies to application code, rewrite intricate T-SQL patterns, and generate test scaffolding.

Rules Engine

Includes SQL Server-aware mappings for `NVARCHAR`, `DATETIME2`, `MONEY`, `HIERARCHYID`, plus collation translation and query pattern detection.

Conversion Modules

Dedicated converters for procedures, triggers, CLR assemblies, SQL Agent jobs, plus `sqlglot`-powered T-SQL to MySQL query rewriting and diffs.

Deep Inventory

Captures memory-optimized tables, temporal tables, FILESTREAM columns, indexed views with `SCHEMABINDING`, permissioned CLR code, and linked servers.

Schema Mapping

Supports flatten, prefix, or database-per-schema strategies to map SQL Server multi-schema estates into TiDB cleanly.
